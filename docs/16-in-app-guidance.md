# In-App Guidance Strategy (Pendo)

## Purpose

This document outlines the in-app guidance strategy for onboarding users, introducing new features, and improving product adoption using digital adoption platforms such as Pendo.

---

# Objectives

The in-app guidance experience is designed to:

- Reduce onboarding time
- Improve feature adoption
- Increase user confidence
- Reduce support requests
- Improve product discoverability

---

# User Journey

```text
First Login
      ↓
Welcome Guide
      ↓
Dashboard Tour
      ↓
Create First Project
      ↓
Configure Settings
      ↓
Execute First Workflow
      ↓
Generate Report
      ↓
Explore Advanced Features
```

---

# In-App Guides

## Welcome Banner

**Trigger:** First login

**Purpose:** Welcome users and introduce the platform.

---

## Dashboard Walkthrough

**Trigger:** First dashboard visit

Highlights:

- Navigation menu
- Projects
- Reports
- Notifications
- User Profile

---

## Feature Tooltip

**Trigger:** First visit to a new feature

Example:

> **Validation Profiles**
>
> Validation Profiles allow you to save and reuse configuration settings across multiple projects.

---

## Checklist

New users complete:

- Create first project
- Configure profile
- Run validation
- Generate report
- Review dashboard

Completion progress is displayed.

---

## Contextual Help

Help icons provide additional guidance for:

- Configuration settings
- Advanced options
- Reports
- API keys

---

# Announcements

Use announcement banners for:

- New releases
- Scheduled maintenance
- Security updates
- New features

---

# Success Metrics

Measure:

- Feature adoption
- Guide completion rate
- Time to first success
- User engagement
- Support ticket reduction

---

# Best Practices

- Keep walkthroughs short.
- Display guidance only when needed.
- Allow users to skip tutorials.
- Avoid interrupting critical workflows.
- Continuously improve guides using analytics.
