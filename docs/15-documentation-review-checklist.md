# Documentation Review Checklist

## Purpose

This checklist is used before publishing documentation to ensure technical accuracy, consistency, usability, accessibility, and release readiness.

---

# Technical Validation

- [ ] Feature implemented
- [ ] Documentation scope complete
- [ ] SME review completed
- [ ] Technical accuracy verified
- [ ] Commands tested
- [ ] API examples validated
- [ ] Screenshots updated

---

# Editorial Review

- [ ] Grammar
- [ ] Spelling
- [ ] Terminology
- [ ] Active voice
- [ ] Consistent formatting

---

# UX Review

- [ ] UI labels verified
- [ ] Navigation correct
- [ ] User workflow complete
- [ ] Cross-links verified

---

# Accessibility Review

- [ ] Alt text added
- [ ] Heading hierarchy correct
- [ ] Tables accessible
- [ ] Color-independent instructions

---

# Release Readiness

- [ ] Release notes completed
- [ ] Version updated
- [ ] Known issues documented
- [ ] Publishing approved
